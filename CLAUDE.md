# CLAUDE.md（リポジトリ全体）

**Claude Code が井上さんの代わりに運用を統括します。**

## 委譲ルール（全セッション共通・厳守）

1. **Claude Code が全部やる。** どんな作業も、まず Claude Code 自身で完結させる。
2. **どうしても Claude Code でできない時だけ、コワークに投げる。** それ以外は委譲しない。
3. **井上さんに返すのは「チェックと承認」だけ。** 選択肢の提示・質問・相談は最小限にする。
4. **不明な点は自分で判断して進める。** 判断できない時だけ一言確認する。
5. **アクセスできない外部URLが来たら、聞かれる前にChrome版用プロンプトを出す。**
   Instagram・ログイン必須サイト・403になるページなど Claude Code から取得できないURLを
   井上さんが投げてきたら、「見れません」で止めずに、**そのままChrome版Claudeに貼れる
   抽出プロンプト**を即生成して提示する。井上さんに二度手間をかけさせない。

主な作業は `ai-sns-automation/` に集約しています。運用の中核ルール・パイプライン・
固定ID・委譲ポリシーは **`ai-sns-automation/CLAUDE.md`** を参照してください。

スマホからの定型操作：
- `/sns` … SNS投稿パイプラインを進める
- `/note-x` … noteの新着記事をXへ投稿する

## インスタ運用チーム（employees/）

インスタ運用を担う6部署が `employees/` 以下に配置されています。
各部署の役割・ルールは各フォルダの `CLAUDE.md` を参照してください。

| 部署 | フォルダ | 役割 |
|---|---|---|
| 秘書 | `employees/secretary/` | タスク管理・部署間連携 |
| リサーチ部 | `employees/research/` | 競合分析・バズ投稿収集 |
| クリエイティブ部 | `employees/creative/` | 投稿テキスト・台本制作 |
| コミュニティ部 | `employees/community/` | コメント文・DM文案 |
| マーケティング部 | `employees/marketing/` | LINE導線設計・ハッシュタグ |
| プロダクト部 | `employees/product/` | 商品企画・教材制作 |

インスタ運用スキル（スマホから呼び出し可）：
- `/週次投稿ネタ出し` … 来週分の投稿ネタを5本生成
- `/フィード投稿生成 テーマ:○○` … テーマ指定でフィード投稿を一括生成
- `/コメント文一括生成` … 今日送るコメント文を一括生成
- `/競合分析 アカウント名:@○○` … 指定アカウントを深掘り分析
- `/週次レポート` … 先週の運用結果を集計
- `/1日まとめ` … 今日の振り返りと明日の準備

秘密情報（APIキー等）は Git に保存せず、環境変数で渡します
（`ai-sns-automation/SETUP_SECRETS.md`）。
