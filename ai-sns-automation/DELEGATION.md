# DELEGATION.md — 委譲ポリシー（Claude Code 主軸）

## 基本構図

**Claude Code が井上さんの代わりに統括する。** まず Claude Code 自身で完結を試み、
処理しきれない時だけ下記の優先順位で委譲する。委譲しても **判断と指示は Claude Code** が行う。

```
[主軸] Claude Code（統括・実行）
   └─ 処理しきれない時 ↓
      1. コワーク      （第1フォールバック / 協働ワークスペース）
      2. Codex         （コード保守・スクリプト修正）
      3. Yoom          （既存の自動トリガー継続）
      4. Chrome版Claude（画面操作の最終手段）
```

> ※「コワーク」の具体サービス・起動方法は環境により異なるため、確定し次第ここに追記する。
> 現状は「Claude Code単体でできない作業の第1委譲先」という位置づけのみ定義する。

## 「Claude Code が処理しきれない」の判断基準

以下に該当する場合に、上から順に委譲先を検討する。

| 状況 | まず試すこと | だめなら委譲 |
|---|---|---|
| 必要なAPI/MCPがある（Drive, Canva, X API等） | Claude Code が直接実行 | — |
| ネットワーク許可が無く外部APIに到達できない | `SETUP_SECRETS.md` の許可ホスト追加を案内 | コワーク / Yoom |
| 公式APIが存在しない画面操作（例: 一部SNSの手動UI） | 代替API・RSS等を探す | Chrome版Claude |
| 大規模なコード改修・既存スクリプトの保守 | Claude Code が実装 | Codex |
| 常時稼働の定期トリガー（人が起動できない時間帯） | （Claude Codeは起動が必要） | Yoom |
| 認証情報が無い／審査が必要（例: Instagram Graph API） | 要件を提示し判断を仰ぐ | Buffer等の既存手段 |

## 委譲時のルール

1. **何を・なぜ委譲するか**を一言で記録（必要ならPR/コメント）。
2. 委譲先が成果を返したら、**結果の検証は Claude Code** が行う。
3. 井上さんの作業を増やさない。確認が必要な時だけ簡潔に尋ねる。
4. 秘密情報は委譲先にも平文で渡さない（環境変数/シークレット参照）。

## 現状の役割対応（旧構成からの移行）

| 旧構成での担当 | 新構成での扱い |
|---|---|
| Yoom（Drive検知→Webhook） | 自動トリガーが必要な場合のフォールバックとして温存 |
| Codex（コード保守の主担当） | Claude Code が主担当。Codexは第2フォールバック |
| Chrome版Claude（画面操作補助） | 最終手段（API/RSSで代替不能な操作のみ） |
| OpenAI画像生成 | Canva MCP に置換（フォールバックとしてOpenAIも可） |
| Buffer投稿 | Instagram等、API直叩きが困難な投稿のフォールバック |
