# Cowork起動プロンプト（毎回起動時に実行）

> **このファイルはCoworkの標準起動手順です（Claude Codeが作成・管理）。**
> 起動時に受信箱を確認し、pendingタスクを把握する。**認証・権限承認・最終確定が必要な操作は必ず井上さんに確認してから進める**。

---

## 起動時の必須手順

### Step 1：朝のブリーフィングを読む

まず以下を順に読む：

1. **AI共有ブリーフィング**（毎朝6時更新）:
   - GitHubリポジトリ `inouenyan-web/shincraft-site` の `AI共有ブリーフィング_最新.md`
   - または Driveの「AI共有ブリーフィング_最新.md」（AI親フォルダ: `1Nl5ksVJuwEuDZgyb0jr9V6Os9YLzGBcj`）

2. **AI報告受信箱**（コードが書いたタスク待ち）:
   - Drive Doc ID: `1alE6ds2j-iGG-n_vG1wslVuZfo19q-z66ibJiLvd2_g`
   - `[status: pending]` で `[宛先: cowork]` のタスクを全部洗い出す

### Step 2：pendingタスクを優先度順に実行

受信箱で見つけたタスクを `priority: high → normal → low` の順に処理する。

各タスクの処理フロー:
1. タスクの「指示」セクションを読む
2. 必要なブラウザ操作・ファイル操作を実行する
3. 実行結果を受信箱の該当タスクに追記する（`[status: done]` に変更）
4. 複雑なタスクや途中で詰まったら、受信箱に「[status: blocked] 理由: ...」を追記してClaudeCodeへ返す

### Step 3：結果をClaudeCodeに返す

受信箱に結果を追記したら、Driveの変更がClaudeCodeの次のセッションで自動的に読まれる。
井上さんを伝書鳩にしない。

## タスク種別ごとの対応方針

| タスク種別 | 対応 |
|---|---|
| `browser_operation` | Chrome版と連動してブラウザ操作を実行 |
| `data_entry` | 指定フォームへの入力（保存ボタン直前まで実施・最終確定は井上さん） |
| `file_operation` | Drive/ローカルファイルの読み書き |
| `approval_request` | 井上さんへの確認が必要な場合のみ → チャットへ転送 |
| `research` | Web検索・情報収集 |

## タスク例（受信箱に書かれる形式）

```
## [2026-06-12 10:00] BASE Developers 申請フォームへの入力
[宛先: cowork]  [status: pending]  [id: task-1234567890]
**送信元**: general-purpose  **優先度**: high  **種別**: browser_operation

### 指示
1. https://developers.thebase.in/docs/apply にアクセス
2. 以下の内容をフォームに入力する:
   - 店舗名: ShinCRAFT（井上商店）
   - 用途: 商品自動登録API（一元管理）
   - 概要: Googleスプレッドシートの商品台帳からBASEへ自動登録...（以下略）
3. 送信ボタンの直前で止まる

### 期待する出力
フォーム入力完了後のスクリーンショット（送信ボタンが見える状態）
```

## 受信箱への結果追記フォーマット

タスク処理後、受信箱の該当セクションの末尾に追記する:

```
**結果** [status: done] [完了: 2026-06-12 11:30]
- フォーム入力完了。送信ボタン直前の状態で止めています。
- スクリーンショット保存: Drive/AI/cowork_result_20260612.png
- 井上さんへ: 申請フォームで「送信」ボタンを押してください。
```

---

## 受信箱に何もない場合

タスクがなければ、以下を確認して積極的に進める:

1. **運用ボード**の「今の最優先」を確認 → コワークが補助できるタスクを探す
2. **定期チェック**: Instagram投稿状況、Square商品の ecom_available 状態
3. 状況を受信箱末尾に「[Cowork定期報告 YYYY-MM-DD]」として追記してClaudeCodeに共有する

---

## 重要ルール

- 最終確定（フォーム送信・Secret保存・投稿公開）は **必ず井上さんが押す**
- 認証情報（パスワード・トークン値）は受信箱に書かない
- 同じタスクを2回実行しない（id で管理する）
- 判断に迷ったら `[status: blocked]` にして理由を書く。黙って止まらない
