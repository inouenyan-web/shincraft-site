# 商品一元管理・EC連携 実装計画（2026-06-08 全員会議の調査結果）

> 調査チーム2班（BASE API班・ZenPlus班）の並行調査結果を統合した実装計画。
> ゴール：**台帳（Googleスプレッドシート）に商品を1行入力 → BASE・ZenPlus・SNS・LINEへ自動展開**

## 全体構成

```
Googleスプレッドシート（商品台帳・既存の投稿管理と同居）
    │
    ├─ Square ─────────── 【最速】MCPコネクタ接続済み（実測確認06-08：井上商店・ACTIVE）
    │                      Claude Codeがセッション内から直接商品登録できる。申請・待ち時間ゼロ
    ├─ BASE API ──────── 商品自動登録（visible=0下書き → 井上さん承認 → 公開）
    │      └─ かんたん海外販売（エントリーだけで100カ国・手数料5%）
    ├─ ZenPlus ────────── CSV一括登録（Claude Codeが変換）→ 海外向け固定URL発行
    ├─ Instagram / X ──── 既存 /sns パイプライン（商品URL・海外URLを自動挿入）
    ├─ LINE公式 ───────── 既存 IG→LINEミラー＋新商品告知
    └─ creema / minne ─── APIなし → 登録用テキスト・画像セットを自動生成（コピペのみ）
```

### Square（2026-06-08 追記・最優先の展開先）

- **Square MCPコネクタがClaude Codeセッションに接続済み**（merchant `井上商店`・JP・ACTIVE を実測確認）。
- カタログ（商品）・在庫・注文・決済の読み書きが**このセッションから直接**できる。
  申請・審査・APIキー登録・Chrome版・井上さんの作業すべて不要。
- 書き込み（商品登録等）は井上さんの確認後に実行する（SNS承認ゲートと同じ運用）。
- **結論：商品一元管理の最初の展開先はSquare**。BASEの審査待ち1〜2週間の間にSquare側を完成させる。

## 調査結果サマリ

### BASE（メインショップ・自動化◎)

- **商品登録API あり**：`POST /1/items/add`（1日1,000件まで）。価格・在庫・バリエーション
  （名入れオーダーの選択肢）・公開/非公開すべてAPIで操作可能。
- **画像**：`add_image` は公開URL方式。Drive直リンク不可 → **GitHub Pages（このリポジトリ）に
  画像を置いてURLを渡す**のが確実。
- **審査が必要**：developers.thebase.in での利用申請に **承認まで1〜2週間**。←最長のリードタイム。
- **Webhookなし**：注文検知は `GET /1/orders` ポーリング or 注文メールのGmail検知で代替。
- **リフレッシュトークン30日**：切れると人間の再認可が必要 → **GitHub Actionsで週1自動リフレッシュ必須**。
- この実行環境から api.thebase.in は現在ブロック → **環境のネットワーク許可に追加が必要**
  （または GAS / GitHub Actions 経由で実行）。

### ZenPlus（越境EC・自動化○）

- **出店：約5分・審査なし・初期/月額0円**。手数料は売れた時の10%のみ。
- 注文後は**大阪倉庫へ国内発送するだけ**。翻訳（19言語/175カ国）・通関・海外発送・
  外国語サポートはZenPlus代行。
- **CSV一括登録（1,000件/回）**：BASE商品データをClaude CodeがZenPlus形式へ自動変換。
- 商品ごとに**固定URL**（例 `https://zenplus.jp/en/{slug}/p/{shop}:{uuid}`）
  → 台帳に「ZenPlus商品URL」列を追加してSNS投稿に自動挿入。
- 全自動化したくなったらネクストエンジン「ZenPlus自動商品登録」（無料アプリ・1日1回自動）へ移行可。

### BASE「かんたん海外販売」（2026-01新機能・作業ゼロ）

- エントリーするだけで海外100カ国以上に販売可能。初期・月額0円、売れた時に決済額の5%。
- ZenPlusと併用可。**エントリー1クリックの価値が大きいので即時有効化推奨**。

### creema / minne（API なし・半自動）

- 自動登録は構造上不可。**登録用テキスト（タイトル・説明・タグ）と画像セットを台帳から
  自動生成**し、井上さんは管理画面にコピペするだけの状態にする。

## 実装フェーズ

### フェーズ0（今すぐ・リードタイム確保）
1. **BASE Developers 利用申請**：承認に1〜2週間かかるため最優先で着手。
   - 申請文面はClaude Codeが作成 → Chrome版が入力 → 井上さんが送信ボタン。
2. **ZenPlus出店申込**（5分）＋ **BASE「かんたん海外販売」エントリー**。
   - どちらも井上さんの確定操作のみ。

### フェーズ1（BASE承認後）
3. OAuth認可フロー構築（コールバックはGAS Webアプリ → 認可コード自動取得・コピペ不要）。
4. トークン自動リフレッシュ（GitHub Actions・週1）。
5. 台帳「商品管理」シート新設 → `items/add`（visible=0）→ 井上さん承認 → visible=1 公開。
   ※SNSパイプラインの承認ゲートと同じ思想。
6. 商品画像のGitHub Pages配置自動化（背景透過済み画像を流用）。

### フェーズ2（BASE商品が載ってから）
7. BASE→ZenPlus CSV変換スクリプト＋台帳へのZenPlus URL記録。
8. /sns 投稿テンプレートに商品URL・海外URL自動挿入。
9. creema/minne 登録テキスト自動生成コマンド。

### フェーズ3（運用安定後・任意）
10. 注文ポーリング→LINE通知。ネクストエンジン移行判断。
