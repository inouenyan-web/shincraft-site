# Yoom設定手順書：ShinCRAFT 01_画像受付→投稿管理登録

## この手順で作るフロー

- フロー名: `ShinCRAFT｜01_画像受付→投稿管理登録`
- 目的: Google Driveの `01_投稿待ち` フォルダに画像が追加されたら、Google Sheetsの `SNS投稿管理台帳_Shincraft` の `投稿管理` シートへ1行追加する
- まず作る範囲: Drive監視からSheets登録まで
- まだ作らない範囲: OpenAI画像生成、投稿文生成、Buffer投稿、SNS投稿結果の反映

## 1. Yoomで新しいフローボットを作る

1. Yoomにログインする。
2. 左側メニューの `マイプロジェクト` を開く。
3. ShinCRAFT用のプロジェクトがある場合はそれを開く。ない場合は任意のプロジェクトでよい。
4. `新規作成` または `フローボットを作成` を押す。
5. テンプレート選択が出た場合は、テンプレートを使わず `はじめから作成` を選ぶ。
6. フローボット名に次を入力する。

```text
ShinCRAFT｜01_画像受付→投稿管理登録
```

## 2. トリガーにGoogle Driveを設定する

Yoom公式ヘルプ上も、Google Drive連携では「特定のフォルダ内に新しくファイル・フォルダが作成されたら」というトリガーが用意されている。

1. 最初のトリガーで `アプリトリガー` を選ぶ。
2. アプリ一覧から `Google Drive` を選ぶ。
3. トリガーアクションで次を選ぶ。

```text
特定のフォルダ内に新しくファイル・フォルダが作成されたら
```

4. Google Driveのアカウント連携画面が出たら、`01_投稿待ち` と `SNS投稿管理台帳_Shincraft` にアクセスできるGoogleアカウントで連携する。
5. トリガーの起動間隔を選ぶ。選択肢がある場合は、まず最短ではなく安定優先で `5分` または `10分` を選ぶ。契約プランで選べる最短間隔が違う場合は、表示される中で運用に合うものを選ぶ。
6. フォルダID入力欄に次を入力する。

```text
17BVeGqN2A7Kj_ppMxGXz-Nejim7UQj0j
```

7. 入力欄が「フォルダURL」形式の場合は次を入力する。

```text
https://drive.google.com/drive/folders/17BVeGqN2A7Kj_ppMxGXz-Nejim7UQj0j
```

8. `テスト` を押す。

### トリガーテスト前に画像を入れる

テストで何も取得できない場合は、Google Driveの `01_投稿待ち` に画像を1枚アップロードしてから、もう一度Yoomの `テスト` を押す。

推奨テストファイル名:

```text
test_shincraft_001.jpg
```

### トリガーテストで確認するアウトプット

テスト成功後、Yoomのアウトプット欄で以下に近い項目が取得できているか確認する。Yoom画面上の日本語名は多少違う場合がある。

| 確認する値 | 使い道 |
|---|---|
| ファイル名 | `商品名` に入れる |
| ファイルID | 必要に応じて確認用に使う |
| ファイルURL / Webリンク | `元画像URL` に入れる |
| 作成日時 | 管理IDや登録日の確認に使う |
| MIMEタイプ | 画像かどうかの確認に使う |

`ファイル名` と `ファイルURL` が取得できていれば、このフローの最初のテストは成功。

## 3. Google Sheetsに行を追加するアクションを設定する

1. Google Driveトリガーの下にある `+` を押す。
2. `アプリと連携する` を選ぶ。
3. アプリ一覧から `Google Sheets` を選ぶ。
4. アクションで、画面に表示される名称に合わせて次のどちらかを選ぶ。

```text
レコードを追加する
```

または

```text
行を追加する
```

5. Google Sheetsのアカウント連携画面が出たら、対象スプレッドシートを編集できるGoogleアカウントで連携する。
6. スプレッドシートIDに次を入力する。

```text
1j8R23sZZfF1h7X1X87EyS1f9KxHkYBPr0ZSbRNxK16s
```

7. タブ名またはシート名に次を入力する。

```text
投稿管理
```

8. テーブル範囲の入力がある場合は、1行目の見出し行を含む範囲を指定する。列がAからQまでの場合は次を使う。

```text
投稿管理!A:Q
```

または、シート名を別欄で指定済みの場合は次を使う。

```text
A:Q
```

9. `次へ` を押し、追加するレコードの値を設定する。

## 4. 列マッピング

Google Sheets側の列名ごとに、次の値を入れる。Yoomのアウトプット挿入欄からGoogle Driveトリガーの値を選ぶ項目は、手入力ではなく必ずYoomの候補から選ぶ。

| Google Sheets列名 | Yoomに設定する値 | 入力方法 |
|---|---|---|
| 管理ID | `SNS-` + `{{実行日時}}` | `SNS-` を手入力し、その後ろにYoomの日時アウトプットを挿入 |
| 登録日 | `{{現在日時}}` | Yoomの日時アウトプットを挿入 |
| 商品名 | `{{Google Driveのファイル名}}` | Google Driveトリガーのアウトプットから挿入 |
| 投稿カテゴリ | `商品紹介` | 固定文字を手入力 |
| 元画像URL | `{{Google DriveのファイルURL}}` | Google Driveトリガーのアウトプットから挿入 |
| 生成画像URL | 空欄 | 何も入力しない |
| Instagram本文 | 空欄 | 何も入力しない |
| X本文 | 空欄 | 何も入力しない |
| CTA | 空欄 | 何も入力しない |
| ハッシュタグ | 空欄 | 何も入力しない |
| 補足メモ | 空欄 | 何も入力しない |
| ステータス | `未確認` | 固定文字を手入力 |
| 投稿予定日 | 空欄 | 何も入力しない |
| Buffer登録結果 | 空欄 | 何も入力しない |
| Instagram投稿URL | 空欄 | 何も入力しない |
| X投稿URL | 空欄 | 何も入力しない |
| エラー内容 | 空欄 | 何も入力しない |

### 管理IDの推奨形式

Yoomの日時フォーマットを指定できる場合は、管理IDを次の形式にする。

```text
SNS-YYYYMMDD-HHmmss
```

例:

```text
SNS-20260525-143012
```

Yoom画面で日時フォーマットを変えられない場合は、まずはYoom標準の実行日時を使ってよい。後で重複が問題になった時点で、Apps Script中継に切り替える。

## 5. Google Sheetsアクションをテストする

1. Google Sheetsアクション画面の `テスト` を押す。
2. 成功表示が出たら、スプレッドシート `SNS投稿管理台帳_Shincraft` を開く。
3. `投稿管理` シートの最終行に1行追加されているか確認する。
4. 追加された行で、最低限以下が入っているか確認する。

| 列名 | 期待値 |
|---|---|
| 管理ID | `SNS-` で始まる値 |
| 登録日 | テスト実行日時に近い日時 |
| 商品名 | `test_shincraft_001.jpg` など、Driveに入れた画像ファイル名 |
| 投稿カテゴリ | `商品紹介` |
| 元画像URL | Google DriveのファイルURL |
| ステータス | `未確認` |

5. テストで入った行が不要な場合は、確認後に手動で削除してよい。

## 6. フローボットを有効化する

1. トリガーとGoogle Sheetsアクションの両方でテストが成功したことを確認する。
2. フローボット編集画面右上の `保存` を押す。
3. フローボットのトグルを `ON` または `起動中` にする。
4. `01_投稿待ち` に別の画像を1枚追加する。
5. トリガーの起動間隔分だけ待つ。
6. `投稿管理` シートに新しい行が追加されることを確認する。

## 7. 本番運用時の入れ方

ユーザーは画像を次のフォルダへ入れる。

```text
01_投稿待ち
https://drive.google.com/drive/folders/17BVeGqN2A7Kj_ppMxGXz-Nejim7UQj0j
```

ファイル名は、そのまま `商品名` に入る。商品名として使いやすい名前にしてからアップロードする。

推奨ファイル名:

```text
商品名_撮影日.jpg
```

例:

```text
ヒノキまな板_20260525.jpg
```

## 8. 失敗時の確認ポイント

### Yoomのトリガーテストでファイルが取れない

- `01_投稿待ち` フォルダIDが `17BVeGqN2A7Kj_ppMxGXz-Nejim7UQj0j` になっているか確認する。
- Yoomに連携したGoogleアカウントで、該当フォルダを開けるか確認する。
- テスト前に画像を1枚アップロード済みか確認する。
- 画像ではなくフォルダを作ってしまっていないか確認する。
- Yoomのトリガーは即時ではなく、起動間隔に従って動く。ON後の本番確認では数分待つ。

### Google Sheetsの設定でシートが見つからない

- スプレッドシートIDが `1j8R23sZZfF1h7X1X87EyS1f9KxHkYBPr0ZSbRNxK16s` になっているか確認する。
- シート名が `投稿管理` になっているか確認する。
- 連携したGoogleアカウントに編集権限があるか確認する。

### 列マッピング画面に列名が出ない

- `投稿管理` シートの1行目に列名が入っているか確認する。
- 空白列や結合セルがないか確認する。
- 範囲指定が `A:Q` または `投稿管理!A:Q` になっているか確認する。
- 列名を修正した後、Yoom側でスプレッドシート情報を再取得する。

### 行は追加されたがURLが入らない

- `元画像URL` に手入力ではなく、Google Driveトリガーのアウトプットを挿入しているか確認する。
- アウトプット候補に `ファイルURL` がない場合は、`Webリンク`、`URL`、`ファイルのリンク` のような名称の項目を選ぶ。
- ファイルIDしか取れない場合は、いったん次の形式でURLを作る。

```text
https://drive.google.com/file/d/{{ファイルID}}/view
```

### 同じ画像で複数行追加される

- Yoomのテストを複数回押すと、そのたびにSheetsへ行が追加される場合がある。
- 本番ON後も重複する場合は、次段階でApps Script中継を使い、ファイルIDの重複チェックを入れる。

## 9. 秘密情報の扱い

- GoogleのOAuth認可はYoomの連携画面で行う。
- APIキー、OAuthトークン、認可コード、Cookieは、このリポジトリにもYoomのメモ欄にも保存しない。
- OpenAIやBufferのAPIキーは今回のフローでは使わない。
- 将来OpenAIやBufferを使う場合も、キーはYoomの接続情報またはサービス側の安全な設定欄に保存し、Markdownファイルには書かない。
