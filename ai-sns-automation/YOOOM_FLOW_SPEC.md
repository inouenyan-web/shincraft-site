# Yoomフロー仕様書（ShinCRAFT SNS自動投稿）

## 目的
Google Driveに入れた写真を起点に、投稿案作成〜承認〜Buffer登録までを自動化する。

## 前提
- 監視フォルダ: `01_投稿待ち`
- 管理台帳: `SNS投稿管理台帳_Shincraft` の `投稿管理` シート
- ステータス: `未確認 / 修正 / 承認 / 投稿予約済み / 投稿済み / エラー`

## フロー1: 写真投入時の受付
**トリガー**
- Google Drive `01_投稿待ち` に新規画像ファイル

**処理**
1. 画像メタ情報取得（ファイル名、URL、作成日時）
2. シート末尾に1行追加
3. `登録日`・`元画像URL`・`ステータス=未確認` を設定
4. 必要に応じて `02_画像生成用元写真` へコピー
5. 次フローへ受け渡し

## フロー2: OpenAIで画像・本文生成
**トリガー**
- フロー1完了

**処理**
1. OpenAI画像生成へ入力（`OPENAI_IMAGE_PROMPT.md`）
2. 生成画像を `03_生成画像` に保存
3. キャプション生成（`OPENAI_CAPTION_PROMPT.md`）
4. `生成画像URL` `Instagram本文` `X本文` `ハッシュタグ` を更新
5. `04_承認待ち` に移動またはコピー

## フロー3: 承認後のBuffer登録
**トリガー**
- シートの `ステータス` が `承認` に変更された行

**処理**
1. 対象行の本文・画像URL・投稿予定日を取得
2. Buffer APIへInstagram/X投稿を登録
3. 成功: `ステータス=投稿予約済み` と `Buffer登録結果` 記録
4. 失敗: `ステータス=エラー` と `エラー内容` 記録

## フロー4: 投稿完了反映（任意）
**トリガー**
- Buffer Webhook / 定期ポーリング

**処理**
1. 投稿済み情報取得
2. `投稿済み` へ更新
3. `Instagram投稿URL` `X投稿URL` を記録
4. ファイルを `05_投稿済み` へ移動

## エラー時の扱い
- 処理失敗行は `ステータス=エラー`
- 原因を `エラー内容` に必ず残す
- 素材は `06_エラー確認` に退避可能にする

## セキュリティ
- APIキー、アクセストークン、OAuth情報はYoom側の秘匿ストアのみ管理
- GitHub / Sheets / Driveファイル本文に秘密情報を書かない
