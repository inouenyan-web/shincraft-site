# Buffer API 連携仕様書

## 目的
承認済み投稿をBufferに登録し、Instagram / Xへの公開スケジュールを管理する。

## 入力元（Google Sheets）
- 生成画像URL
- Instagram本文
- X本文
- ハッシュタグ
- 投稿予定日

## 出力先（Google Sheets）
- Buffer登録結果
- ステータス（投稿予約済み / エラー）
- 投稿URL（取得できる場合）

## 連携時の基本要件
1. `ステータス=承認` の行のみ対象
2. 投稿先チャネル（Instagram, X）を明示
3. 投稿予定日が未設定の場合は即時投稿ではなくエラーにする（運用事故防止）
4. API成功時はレスポンスIDを `Buffer登録結果` に記録
5. API失敗時はエラー本文を `エラー内容` に記録

## 画像URL要件
- Buffer APIからアクセス可能な公開URLであること
- Google Driveの通常共有URLは失敗する場合があるため、公開URL化手順をYoom側で持つこと

## リトライ方針
- 429 / 5xx は指数バックオフで最大3回
- 4xx（認証/権限/入力不正）は即エラー終了

## セキュリティ
- Buffer APIキーはYoomのシークレット管理で保持
- APIキーをコード、シート、Markdownに直書きしない
