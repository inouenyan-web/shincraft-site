# Yoom次段階フロー設計

## 全体方針

最初に確実に動かす範囲は、Google Driveの `01_投稿待ち` に画像が追加されたら、Google Sheetsの `投稿管理` シートへ1行追加するところまで。

その後のOpenAI画像生成、投稿文生成、承認、Buffer投稿は、Sheetsの `ステータス` を軸に段階的に追加する。

## フロー1: 画像受付→Sheets登録

- 状態: 今回設定する
- Yoomフロー名: `ShinCRAFT｜01_画像受付→投稿管理登録`
- トリガー: Google Drive `01_投稿待ち` にファイル・フォルダが作成されたら
- 監視フォルダID: `17BVeGqN2A7Kj_ppMxGXz-Nejim7UQj0j`
- アクション: Google Sheets `投稿管理` に1行追加
- 初期ステータス: `未確認`

登録する主な値:

| 列 | 値 |
|---|---|
| 管理ID | `SNS-{{実行日時}}` |
| 登録日 | `{{現在日時}}` |
| 商品名 | Driveのファイル名 |
| 投稿カテゴリ | `商品紹介` |
| 元画像URL | DriveのファイルURL |
| ステータス | `未確認` |

## フロー2: OpenAI画像生成

- 状態: 次段階
- 起動条件: `投稿管理` シートで `ステータス = 画像生成待ち` の行を対象にする
- 入力: `元画像URL`、`商品名`、`投稿カテゴリ`
- 出力: 生成画像をGoogle Driveの `03_生成画像` に保存
- 保存先フォルダID: `1eVUuN8qYuHp7h0h_I13nNS6TwuiHRcT7`
- Sheets更新:
  - `生成画像URL`
  - `ステータス = 本文生成待ち`
  - 失敗時は `ステータス = エラー`、`エラー内容` に理由

注意:

- OpenAI APIキーはリポジトリに保存しない。
- Yoomの接続情報または安全な環境変数設定欄に保存する。
- 画像URLが外部から取得できない場合は、Apps ScriptやDrive共有設定を検討する。

## フロー3: OpenAI投稿文生成

- 状態: 次段階
- 起動条件: `ステータス = 本文生成待ち`
- 入力: `商品名`、`投稿カテゴリ`、`元画像URL`、`生成画像URL`、必要に応じて `補足メモ`
- 出力:
  - `Instagram本文`
  - `X本文`
  - `CTA`
  - `ハッシュタグ`
- Sheets更新:
  - `ステータス = 承認待ち`

投稿文生成時の原則:

- Instagram本文は読みやすさ優先。
- X本文は文字数を短めにする。
- ハッシュタグは商品・ブランド・素材・用途を混ぜる。
- 本文生成後、すぐ投稿せず必ず承認待ちにする。

## フロー4: 生成結果をSheetsへ戻す

- 状態: 次段階
- 起動条件: フロー2またはフロー3の完了
- 更新対象: `管理ID` で行を特定する
- 更新内容:
  - `生成画像URL`
  - `Instagram本文`
  - `X本文`
  - `CTA`
  - `ハッシュタグ`
  - `ステータス`
  - `エラー内容`

実装メモ:

- Yoomだけで行検索と更新が複雑になる場合は、Apps Script中継を使う。
- `管理ID` は全フローで共通キーとして扱う。

## フロー5: 承認後にBufferへ投稿予約

- 状態: 次段階
- 起動条件: `ステータス = 承認済み`
- 追加条件:
  - `投稿予定日` が空欄でない
  - `Instagram本文` または `X本文` が空欄でない
  - `生成画像URL` が空欄でない
- アクション: Bufferに投稿予約
- Sheets更新:
  - 成功時: `Buffer登録結果 = 成功`
  - 成功時: `ステータス = 投稿予約済み`
  - 失敗時: `Buffer登録結果 = 失敗`
  - 失敗時: `ステータス = エラー`
  - 失敗時: `エラー内容` に理由

注意:

- BufferのAPIキーやOAuthトークンはリポジトリに保存しない。
- 投稿予約前に必ず人間が `承認済み` に変更する。

## フロー6: 成功/失敗をSheetsへ反映

- 状態: 次段階
- 起動条件: Buffer投稿結果または手動確認
- 更新内容:
  - `Instagram投稿URL`
  - `X投稿URL`
  - `Buffer登録結果`
  - `ステータス`
  - `エラー内容`
- 成功時ステータス: `投稿済み`
- 失敗時ステータス: `エラー`

## 推奨ステータス一覧

| ステータス | 意味 |
|---|---|
| 未確認 | Driveから登録された直後 |
| 画像生成待ち | 人間が画像生成に進めてよいと判断した状態 |
| 本文生成待ち | 画像生成が完了し、投稿文生成待ち |
| 承認待ち | 投稿画像と本文が揃い、人間の確認待ち |
| 承認済み | Buffer投稿予約に進めてよい状態 |
| 投稿予約済み | Buffer登録が完了した状態 |
| 投稿済み | SNSへの投稿完了を確認済み |
| エラー | どこかの処理で失敗し確認が必要 |

## 次に作るべき順番

1. `ShinCRAFT｜01_画像受付→投稿管理登録` を本番ONにする。
2. 実画像を数枚入れて、Sheets登録が安定するか確認する。
3. 重複登録が発生するか確認する。
4. 重複がなければ、Yoom標準機能でフロー2へ進む。
5. 重複や行更新の複雑さが出たら、Apps Script中継を先に作る。
