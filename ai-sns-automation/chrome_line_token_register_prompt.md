# Chrome版Claudeへの指示：LINEトークンを Claude Code 環境変数に登録（確定の直前まで）

以下を **Claude for Chrome 拡張機能** にそのまま貼り付けてください。
Chrome版が **入力までを全部やり、井上さんは最後の「保存／確定」を押すだけ** の状態にします。

> ゴール：Claude Code on the web の環境変数設定画面を開き、
> `LINE_CHANNEL_ACCESS_TOKEN` の名前と値を入力し終えた状態にする。
> **保存ボタンは押さず、井上さんが内容を確認して確定できる状態で止める。**

---

## 拡張へのプロンプト（コピペ用）

```
Claude Code on the web の「環境変数（Environment variables）」設定画面で、
新しい環境変数を1つ、入力するところまで進めてください。保存は押さず、入力完了状態で止めてください。

【1. 設定画面を開く】
- https://claude.ai/code を開く（ログイン済み前提。未ログインなら井上さん本人のログインが必要）
- 対象の環境（shincraft-site のリモート実行環境）の設定 → 「環境変数 / Environment variables」を開く
  ※メニュー名が「Settings」「Environment」「変数」等の場合あり。環境変数を追加できる欄を探す。

【2. 新しい変数を追加（入力のみ）】
- 「変数を追加 / Add variable」をクリック
- 名前（Name / Key）欄に、以下を正確に入力：
    LINE_CHANNEL_ACCESS_TOKEN
- 値（Value）欄に、LINE Developers で取得済みの「チャンネルアクセストークン（長期）」全文を
  貼り付け（前後の空白・改行を入れない）。
  ※トークンは秘密情報のためこのファイルには書かない。井上さん本人が手元の値を貼る／
    Chrome版が LINE Developers 画面からコピーして貼る。

【3. ★ここで止める（重要）】
- 「保存 / Save / 追加確定」ボタンは**押さない**。
- 名前と値が正しく入力された状態のまま、画面を井上さんに見せて
  「内容を確認して保存ボタンを押してください」と伝える。

【4. 報告】
- 入力が完了して保存待ちの状態になったら、その旨を報告。
- もし環境変数の追加欄が見つからない／画面に到達できない場合は、
  どこで詰まったか（画面名・表示メッセージ）を具体的に報告。
```

---

## 井上さんがすること（これだけ・本人のみ可能）

Chrome版が名前と値を入力し終えた画面で、**内容を確認して「保存」ボタンを押すだけ。**

> なぜ井上さん本人か：Claude Code の環境変数の**最終確定**は本人の操作が要るため
>（CLAUDE.md の「本人にしかできない操作」#3）。入力作業はChrome版が代行する。

保存後は、次の `/sns` → 承認 → 投稿 のタイミングで、Instagram投稿と同時に
LINE公式アカウントの全フォロワーにも自動でブロードキャストされます。
コードは実装・テスト済み（`scripts/lib/line_client.mjs` ／ `post_to_buffer.mjs`）。

## もし Chrome版で環境変数画面に到達できなかった場合

その時だけ、井上さん本人が同じ画面を開いて、上記【2】の名前と値を貼り付けて保存してください。
（Chrome版が「画面に到達できない」と報告してきた場合のフォールバック）
