# /chrome-handoff — 連携・同期レーン（Chromeハンドオフ）

## このスキルの使いどき
- ログイン必須のWeb画面操作（X投稿・note下書き保存・Instagram画面確認等）
- Claude Code のネット制限で直接到達できないホストへのアクセス
- API が無い or ブロックされているサービスへの操作

## 使う前の自問（毎回確認）
> **「井上さんが直接やった方が速くないか？」**
> URLを開いて1〜2手で完了するなら Chrome版 を経由するより本人が直接やる方が速い。
> Chrome版が本当に必要なのは「ログイン状態を保ったままの長い画面操作・スクレイピング」に限る。

## 手順

### 1. 渡す内容を整理する
- 何をしてほしいか（操作の目的・URL・入力値）
- 取得してほしい情報がある場合はその形式（テキスト/スクショ/JSON等）

### 2. Chrome版Claude向けプロンプトを生成する
下記テンプレートに埋めて出力する。

```
## Chrome版Claude へのプロンプト（コピペ用）

### 目的
{1行で操作の目的}

### 手順
1. {URL} を開く
2. {操作内容（ログイン済み前提）}
3. {取得 or 入力 or 確認したい内容}

### 返してほしいもの
{テキスト / スクショ / 値のコピー等}
```

### 3. 本人への提示
- 「このプロンプトをChrome版Claudeに貼ってください」と伝える
- プロンプトは貼るだけで動く状態にする（値の穴埋めを本人に頼まない）

## X/note 投稿の場合
- 発信（note/X）レーンで `/note` または `/note-x` を先に試す
- APIが止まっている・ログインが要る場合にのみここへ落とす

## 環境変数・設定変更の依頼が来た場合
- Claude Code の環境変数変更 → Chrome版では不可。**本人がWeb設定画面で行う**
- GitHub Secrets → 同上
- これらは本人にしかできない操作として案内する
