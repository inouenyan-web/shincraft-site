# /競合分析

引数 `$ARGUMENTS` でアカウント名を受け取る（例：`アカウント名:@example_account`）。

## 手順

1. 指定アカウントの直近30投稿の情報を整理（フィード・リール・ストーリーズの構成比）
2. 各投稿の保存数・コメント数・いいね数を取得（分かる範囲で）
3. 上位5投稿の共通点を抽出（テーマ・構成・ビジュアル・投稿時間）
4. このアカウントから学ぶべきポイントを3つ挙げる
5. ShinCRAFTのアカウントに応用する案を5つ出す
6. 結果を `employees/research/competitors/{account_name}.md` に保存

## 出力フォーマット

```markdown
# 競合分析：@{account_name}

分析日：YYYY-MM-DD
フォロワー数：
ジャンル：

## 投稿構成比（直近30投稿）
- フィード：X投稿
- リール：X投稿
- ストーリーズ：確認不可

## 上位5投稿の共通点
| 順位 | テーマ | 構成 | ビジュアル特徴 | 投稿時間帯 |
|-----|--------|------|-------------|---------|
| 1   |        |      |             |         |

## このアカウントから学ぶポイント
1. 
2. 
3. 

## ShinCRAFTへの応用案
1. 
2. 
3. 
4. 
5. 
```
