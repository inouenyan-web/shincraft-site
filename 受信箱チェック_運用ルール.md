# 受信箱チェック 固定運用ルール（受注見落とし防止）

このファイルは、Gmail / Chatwork / Instagram の受注関連を取りこぼさないための
**固定の運用指示**です。コード（GitHub Actions）と Cowork（スケジュールタスク）の
両方で、**1日4回（日本時間 0 / 6 / 12 / 18 時）** に回します。

---

## 1. 自動チェック（GitHub Actions: daily-briefing.yml / 1日4回）

毎回の実行で以下を行い、結果を `運用ボード.md` に追記する：

- **Gmail**（`scripts/check_gmail.mjs`）
  - 直近24hの未読から受注関連（注文/見積/納期/在庫/名入れ 等）を抽出 → タスク化
  - 認証: `GMAIL_CLIENT_ID` / `GMAIL_CLIENT_SECRET` / `GMAIL_REFRESH_TOKEN`（shincraft2023）
- **Chatwork**（`scripts/check_chatwork.mjs`）
  - **既読ルームのみ**本文をスキャンして受注関連を抽出（未読は消費しない＝既読化しない）
  - 未読ルームは「未読あり」件数のみ記録し、手動確認を促す
  - 認証: `CHATWORK_API_TOKEN`
- **Instagram DM リマインド**（`scripts/instagram_dm_reminder.mjs`）
  - DM自動取得が未完了の間は、手動チェックのリマインド＋Chrome用プロンプトを表示
- **Instagram 投稿/コメント/返信漏れ**（`instagram-check.yml` / 1日4回）

cron（UTC）: `0 15,21,3,9 * * *`（= JST 0/6/12/18時）

---

## 2. Cowork 固定タスク（scheduled task: shincraft-inbox-4x / 1日4回）

`運用ボード.md` の受注関連タスクをまとめて井上さんに報告し、
InstagramのDMは手動確認を促す。トークン健全性（下記）も毎回チェック。

---

## 3. Instagram DM 自動化（要セットアップ：本人操作が1回だけ必要）

DMは「Instagram API with Instagram **Business Login**」方式で取得する。

- 必要スコープ: `instagram_business_basic`, **`instagram_business_manage_messages`**
  （※ `instagram_manage_messages`（business_なし）はFacebookログイン方式用で**使わない**）
- App Review・ビジネス認証は**不要**（自分アカウントのDM読み取りは Standard Access）。開発モードのままで可。
- Instagram App ID: `833113012864558`（アプリ名 shincraft-check-IG）
- **本人操作が必須なのは「初回トークン発行（Instagramログイン＋許可）」の1回だけ。**
  Meta開発者コンソール → Instagram → API setup with Instagram business login →
  対象アカウントの「Generate token」をクリックして発行（実質ワンクリック）。
- 取得後の自動化（コード側）:
  - DM会話一覧: `GET https://graph.instagram.com/v25.0/me/conversations?platform=instagram`
  - メッセージ本文: `GET https://graph.instagram.com/<MESSAGE_ID>?fields=id,created_time,from,to,message`
  - **本文は直近20件まで**しか取れない → 取りこぼし防止のため頻繁にポーリング（1日4回でも不足する可能性あり、要監視）
  - 長期トークン（60日）は `GET https://graph.instagram.com/refresh_access_token?grant_type=ig_refresh_token` でサーバー側のみで自動更新
- 失効条件（要再ログイン＝本人操作）: パスワード変更 / 権限取消 / 60日放置

### トークン健全性チェック（毎回）
- DM自動化が有効な場合、トークン更新失敗・失効の兆候がないか確認し、異常時は井上さんへ再ログインを依頼する。

## 4. 受注タスクの扱い
- 受注関連は `運用ボード.md` の「進行中タスク」直前に `- [ ]` 形式で並べる。
- 「未返信・期限が近い・要対応」は最優先として先頭に強調。
